1st Day: All the Syntax for GitHub

What is Git Bash?

Git is a version control system used by developers to track changes in code, manage different versions, and collaborate with others on projects.

Git Bash is a command-line tool for Windows that allows you to use Git commands and Linux-style terminal commands in one place.

We use Git Bash in our daily development work to manage files and repositories. It helps developers create repositories on GitHub, create branches, and work on different features without affecting the main project. Multiple developers can work on different branches and later merge their changes into the main branch safely.

To configure Git Bash in our coding environment like VS Code, we can use the following steps:

Step 1: First download Git for your system using this link: Download Git for Windows

Step 2: Open Git Bash and enter the following commands to setup Git.

  • git config --global user.name "Your Name" : Sets your username (same as your GitHub username).
  • git config --global user.email "youremail@gmail.com" : Sets your email (same as your GitHub account email).
  • git config --list : Checks whether the configuration was completed successfully.

How to Setup Git in VS Code

Step 3: Create a folder on your system (example: Test.git)

Step 4: Open the folder in VS Code and run the following commands in the terminal.

  • git init : Initializes a repository inside the folder.
  • git clone repository_url : Downloads an existing repository from GitHub.

Step 5: Git File Stages

  • git status : Shows the current project status of all files.
  • Stage 1 – Untracked : Git does not know about the file yet.
  • Stage 2 – Staged : File is ready to be committed.
    Command: git add filename
    Example: git add index.html
    Add all files: git add .
  • Stage 3 – Modified : File was committed before but changed again.
  • Stage 4 – Commit : Changes are permanently saved in repository history.
    Command: git commit -m "descriptive message"

Step 6: Check History and Changes

  • git log : Shows complete commit history with commit IDs.
  • git show commitID : Shows changes made in a specific commit.

Step 7: Create a new repository on GitHub.

Step 8: Connect Local Repository to GitHub Repository

  • git remote add origin repoURL : Connects local repository to GitHub.
  • git remote -v : Shows connected repositories.
  • git remote remove origin : Disconnects the repository.
  • git push origin main : Uploads files to GitHub repository.
  • git push origin branch-name : Pushes code to a specific branch.
  • git pull origin main : Downloads latest changes from GitHub.

Step 9: Creating Branches

  • git branch : Shows list of branches.
  • git branch branchName : Creates a new branch.
  • git checkout branchName : Switches to another branch.

Step 10: Merge Branches

  • git merge branchName : Merge another branch into current branch.
  • git merge --abort : Abort merge if conflict occurs.
Merge Conflict Example

If two branches change the same code location, Git will show something like this: 


<<<<<<< HEAD
Your current branch code
=======
Other branch code
>>>>>>> feature-branch

After fixing the conflict:

  • git add filename
  • git commit

Using these steps you can create, modify, and upload files to your GitHub account.
